General Checks - Playwright Audit Report

This report analyzes all 18 general checks. For each check, we consider what the UI FindingCard strictly requires, identify if Playwright is instantiated, and determine if Playwright is absolutely not needed.

1) Privacy Policy (privacy_policy) Playwright: Present
Verdict: Needed

Analysis: The PrivacyPolicyFindingCard requires visual proof screenshots of the footer (containing the privacy link), the checkout page (for WooCommerce notices), and the full policy page. Navigating to multiple paths, evaluating the DOM, and capturing accurate screenshots requires Playwright.

2) Project Plan (project_plan) Playwright: Present
Verdict: Needed

Analysis: The ProjectPlanFindingCard explicitly requires and renders two screenshots: "Plan Highlight" and "Reviews Page". While the plan highlight is generated from Basecamp text, the check must navigate to the live /reviews page to capture a viewport screenshot. Because this screenshot from the actual website is required by the UI, Playwright is absolutely needed here.

3) Paid Media (paid_media) Playwright: Not Present
Verdict: Not needed

Analysis: The PaidMediaFindingCard displays data based on scanning Basecamp APIs for campaign mentions. No UI screenshot or browser interaction is required. It correctly uses axios instead of Playwright.

4) Hero Media (hero_media) Playwright: Present
Verdict: Needed

Analysis: The HeroMediaFindingCard relies on metrics for video load times and fallback image validation. Playwright is required to evaluate browser-level events (like the playing event on a <video> element) and to capture targeted fallback/hero screenshots for visual inspection.

5) Footer Logo (footer_logo) Playwright: Present
Verdict: Needed

Analysis: The LogoCheckFindingCard explicitly requires multi-viewport screenshots (Desktop, Tablet, Mobile) of the footer section so users can manually verify if a tagline is present. Capturing and isolating the footer element accurately necessitates Playwright.

6) Single Script (single_script) Playwright: Present
Verdict: Needed

Analysis: The SingleScriptFindingCard needs screenshots of the injected #feature-buttons widget across all viewports to verify visual injection and alignment, plus a code snippet screenshot. Playwright is required for layout rendering and capturing.

7) Top Bar & Sticky Header (top_bar_sticky) Playwright: Present
Verdict: Needed

Analysis: The HeaderFindingCard visually verifies the layout and presence of social/contact links inside the top bar. Playwright is needed to capture the header element screenshot accurately.

8) Favicon (favicon) Playwright: Present
Verdict: Needed

Analysis: The FaviconFindingCard requires a screenshot of a mocked browser tab UI displaying the injected favicon, along with the code snippet. This visual proof requires Playwright's rendering engine and DOM evaluation.

9) Chatbot Consultation (chatbot_consultation) Playwright: Present
Verdict: Needed

Analysis: While the FindingCard doesn't explicitly display a screenshot, the check relies on simulating a physical click() event on the launcher button and verifying if the chatbot window transitions to a visible state. This interactive testing is impossible with static HTML parsers and requires Playwright.

10) Text Share (text_share) Playwright: Present
Verdict: Not needed

Analysis: The FindingCard only needs the string values of the Open Graph metadata (og:title, og:site_name, twitter:title) to check for WordPress boilerplate. No screenshots are taken. A simple, much faster HTTP GET using axios and cheerio would easily suffice here. Playwright is absolutely not needed.

11) Learn More Buttons (learn_more_buttons) Playwright: Not Present
Verdict: Not needed

Analysis: The LearnMoreButtonsFindingCard lists elements containing generic CTA text. It intelligently uses got and cheerio to parse the DOM without needing Playwright.

12) URL & Tab Compare (url_tab_compare) Playwright: Not Present (Imported but unused)
Verdict: Not needed

Analysis: The check logic crawls sitemaps and compares dev vs live URLs. The FindingCard parses a JSON payload to display a comparison table. It uses axios and cheerio under the hood. Playwright is not needed.

13) Verify Plugin Updates (verify_plugin_updates) Playwright: Present
Verdict: Needed

Analysis: The PluginUpdatesFindingCard requires a screenshot of the WordPress plugins dashboard. Playwright is needed to bypass login screens, navigate the WP admin area, and screenshot the plugin list.

14) Social Share Heading (social_share_heading) Playwright: Present
Verdict: Needed

Analysis: The SocialShareHeadingFindingCard displays screenshots generated by an external tool (socialsharepreview.com). Navigating to that site, submitting the form, and capturing screenshots of the Facebook, X, and LinkedIn tabs mandates Playwright.

15) Logo on Chatbot (logo_chatbot) Playwright: Present
Verdict: Needed

Analysis: The LogoOnChatbotFindingCard requires screenshots of the homepage and the explicitly opened chatbot window to verify branding. Triggering the chatbot to open and capturing the result requires Playwright.

16) Contact Form (contact_form) Playwright: Present
Verdict: Needed

Analysis: The ContactFormFindingCard requires visual proof of the form. Furthermore, the check actively fills out the form in an iframe, submits it, and captures the "Thank You" screen. Playwright is strictly necessary for this interaction.

17) Dead Links (dead_links / broken_links) Playwright: Present
Verdict: Not needed

Analysis: The DeadLinksFindingCard outputs a text list of broken HTTP response codes and uses a pre-existing full-page screenshot (pageRecord.desktopUrl) as its generic image. Currently, Playwright is used to extract <a href> tags from the DOM, but this could easily be achieved with axios and cheerio. Playwright is absolutely not needed here.

18) Callnow Links (callnow_links) Playwright: Present
Verdict: Needed

Analysis: The CallnowFindingCard requires visual proof of the Call Now button plugin setup inside the WordPress admin panel, as well as a mobile-viewport screenshot of the homepage showing the button. Playwright is necessary for the authentication, navigation, and specific viewport captures.
